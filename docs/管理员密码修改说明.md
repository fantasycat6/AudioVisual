# 管理员密码修改功能说明

## 问题描述
用户反馈管理后台中管理员无法修改自己的密码，只能重置普通用户的密码。

## 原因分析
在原有的用户管理界面中，为了保护管理员账户安全，当前登录的管理员在用户列表中显示为"(当前用户)"，没有提供操作按钮。这种设计虽然增强了安全性，但导致管理员无法修改自己的密码。

## 解决方案
我们新增了一个专门的"修改密码"功能，允许管理员在验证当前密码后修改自己的密码。

### 功能特性

#### 1. 界面改进
**位置**: 管理后台 → 用户管理

**改动**:
- 当前登录的用户在用户列表中会看到一个"修改密码"按钮（而不是之前的"(当前用户)"文本）
- 其他用户不受影响，仍然显示操作按钮

#### 2. 密码修改流程
1. 点击"修改密码"按钮
2. 弹出修改密码模态框，包含三个输入框：
   - **旧密码**: 当前正在使用的密码
   - **新密码**: 至少6个字符的新密码
   - **确认新密码**: 再次输入新密码进行确认
3. 点击"确认修改"按钮提交

#### 3. 安全验证
- **旧密码验证**: 必须提供正确的当前密码
- **新密码长度**: 至少6个字符
- **密码一致性**: 新密码和确认密码必须一致

## 技术实现

### 前端 (`templates/admin/users.html`)
```html
<!-- 在用户操作列中 -->
{% if user.id != current_user.id %}
  <!-- 原有操作按钮 -->
{% else %}
  <button class="btn btn-ghost btn-xs" onclick="showChangePasswordModal()">
    修改密码
  </button>
  <span class="muted-text">（当前用户）</span>
{% endif %}

<!-- 新增的修改密码模态框 -->
<div id="change-password-modal" class="modal-overlay" style="display:none;">
  <!-- 模态框内容 -->
</div>
```

### 后端 (`admin.py`)
```python
@admin_bp.route('/users/change-password', methods=['POST'])
@login_required
@admin_required
def change_password():
    """修改当前用户的密码"""
    data = request.get_json()
    old_password = data.get('old_password', '').strip()
    new_password = data.get('new_password', '').strip()
    
    # 验证旧密码
    if not current_user.check_password(old_password):
        return jsonify({'success': False, 'message': '旧密码错误'}), 400
    
    # 验证新密码
    if len(new_password) < 6:
        return jsonify({'success': False, 'message': '新密码至少需要6个字符'}), 400
    
    # 更新密码
    current_user.set_password(new_password)
    db.session.commit()
    
    return jsonify({'success': True, 'message': '密码修改成功'})
```

## 权限说明

### 两种密码操作模式
1. **管理员重置其他用户密码** (无需验证旧密码)
   - 适用于用户忘记密码的情况
   - 管理员有权限直接为用户设置新密码

2. **当前用户修改自己密码** (需要验证旧密码)
   - 适用于用户主动更改密码
   - 增强安全性，防止账户被未授权修改

## 使用指南

### 1. 普通用户密码重置
1. 登录管理员账户
2. 进入"管理后台" → "用户管理"
3. 找到需要重置密码的用户
4. 点击"重置密码"按钮
5. 输入新密码即可

### 2. 管理员修改自己密码
1. 登录管理员账户
2. 进入"管理后台" → "用户管理"
3. 找到自己的用户名（列表中会显示"(当前用户)"标记）
4. 点击新出现的"修改密码"按钮
5. 输入当前密码和新密码
6. 确认修改

## 错误处理

### 常见错误及解决方案
| 错误情况 | 原因 | 解决方案 |
|---------|------|---------|
| "旧密码错误" | 输入的当前密码不正确 | 检查密码是否正确，注意大小写 |
| "新密码至少需要6个字符" | 新密码太短 | 使用至少6个字符的密码 |
| "新密码和确认密码不匹配" | 两次输入的新密码不一致 | 确保两次输入的密码完全相同 |
| "旧密码和新密码不能为空" | 有字段为空 | 填写所有必填字段 |

## 安全建议

### 1. 密码策略
- 建议使用强密码：包含大小写字母、数字和特殊字符
- 避免使用容易猜到的密码（如123456、password等）
- 定期更换密码（建议每3-6个月更换一次）

### 2. 账户安全
- 确保管理员账户使用安全密码
- 避免在公共计算机上保持登录状态
- 启用浏览器安全特性

## 向后兼容性
- 原有功能完全保留：管理员仍然可以为其他用户重置密码
- 新增功能不影响已有功能
- 所有API接口保持向下兼容

## 测试验证
1. 可以成功修改当前登录管理员的密码
2. 修改后可以使用新密码登录
3. 旧密码不再有效
4. 未登录用户无法访问此功能
5. 非管理员用户也无法访问此功能

## 故障排除
如遇问题，请检查：
1. 确保使用正确的旧密码（区分大小写）
2. 确保新密码长度足够（≥6字符）
3. 确保两次输入的新密码一致
4. 检查浏览器控制台是否有JavaScript错误
5. 查看服务器日志是否有后端错误
