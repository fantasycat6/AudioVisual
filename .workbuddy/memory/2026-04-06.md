## Session持久化优化

### 问题描述
用户报告服务器重启后登录状态会丢失，每次都需要重新登录，用户体验不友好。

### 解决方案
实现了session持久化存储，使用session-file-store替代默认的MemoryStore。

### 实施步骤
1. **安装session-file-store依赖**：
   ```
   npm install session-file-store
   ```

2. **修改app.js**：
   - 引入FileStore模块
   - 配置session使用文件存储
   - 设置存储目录为`./data/sessions/`
   - 配置30天存储有效期和自动清理机制
   - 移除全局cookie.maxAge设置，让登录逻辑决定有效期

3. **创建必要目录**：
   - 确保`data/sessions/`目录存在

4. **保持现有登录逻辑**：
   - "记住我"功能继续正常工作
   - 勾选"记住我"：30天有效期，浏览器会话结束后保持登录状态
   - 未勾选"记住我"：会话cookie，浏览器关闭后过期

### 技术配置
```javascript
// app.js 中的session配置
app.use(session({
    store: new FileStore({
        path: './data/sessions',
        ttl: 30 * 24 * 60 * 60,  // 30天存储
        retries: 5,
        reapInterval: 3600,
        reapMaxConcurrent: 1
    }),
    secret: process.env.SESSION_SECRET || 'audiovisual-secret-key-2026',
    resave: false,
    saveUninitialized: false,
    cookie: {
        secure: process.env.NODE_ENV === 'production',
        httpOnly: true
    }
}));
```

### 验证方法
1. 登录时勾选"记住我"
2. 检查`data/sessions/`目录下生成session文件
3. 停止服务器
4. 重新启动服务器
5. 刷新页面，用户应保持登录状态

## Favicon 修复 - 后台管理页面

### 问题发现
用户报告后台管理的数据备份与恢复页面没有正确显示favicon.ico图标。

### 原因分析
虽然favicon文件存在（`./public/favicon.ico`）且路径正确，但在某些深度路由页面（如`/admin/backup`）中，路径解析可能存在问题。

### 解决方案
1. **添加根路径favicon路由**（`src/app.js`）：
   ```javascript
   app.get('/favicon.ico', (req, res) => {
       res.sendFile(path.join(process.cwd(), 'public/favicon.ico'));
   });
   ```

2. **更新favicon引用路径**（`src/views/partials/header.ejs`）：
   - 将 `/public/favicon.ico` 改为 `/favicon.ico`

### 技术优势
- **标准化**: 使用根路径确保在所有路由页面都能正确解析
- **兼容性**: 保留原有的 `/public/favicon.ico` 访问路径，向后兼容
- **可靠性**: 明确的文件发送路由，减少路径解析问题

### 验证方法
1. 访问 `http://localhost:5010/favicon.ico` 应返回200状态码和图标文件
2. 访问 `http://localhost:5010/admin/backup` 应正确显示favicon

## 登录问题修复 - 2026-04-07

### 问题发现
用户报告 `/login?next=%2F` 无法登录。

### 根本原因分析
1. **Flash消息中间件问题**：
   - 原有的flash中间件在每次请求开始时就把`req.session.flash`清空
   - 导致登录页面无法接收到错误消息或成功消息
   
2. **Flash API不一致**：
   - 代码中同时使用了`req.flash`和`req.session.flash`两种形式
   - 但中间件只设置了`req.session.flash`，没有设置`req.flash`方法

### 解决方案
在`src/app.js`中完全重写flash中间件逻辑：

1. **修改前**：
```javascript
// Flash 消息中间件
app.use((req, res, next) => {
    res.locals.flash = req.session.flash || {};
    // ...
});

// 清除 flash 消息  
app.use((req, res, next) => {
    req.session.flash = {};
    next();
});
```

2. **修改后**：
```javascript
// Flash 消息中间件 (修复版)
app.use((req, res, next) => {
    // 初始化session.flash数组
    if (!req.session.flash) {
        req.session.flash = { error: [], success: [], info: [] };
    }
    
    // 创建req.flash方法用于向后兼容
    req.flash = {};
    ['error', 'success', 'info'].forEach(type => {
        req.flash[type] = function(message) {
            if (typeof message === 'string') {
                req.session.flash[type].push(message);
            }
            return this;
        };
    });
    
    // 设置res.locals变量供EJS模板使用
    res.locals.flash = {
        error: req.session.flash?.error?.[0] || null,
        success: req.session.flash?.success?.[0] || null,
        info: req.session.flash?.info?.[0] || null
    };
    
    next();
});

// 在响应结束后清除flash消息
app.use((req, res, next) => {
    const originalEnd = res.end;
    res.end = function(...args) {
        // 只在响应结束时清除flash
        if (req.session.flash) {
            req.session.flash.error = [];
            req.session.flash.success = [];
            req.session.flash.info = [];
        }
        return originalEnd.apply(this, args);
    };
    next();
});
```
