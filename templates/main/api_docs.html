📚 API 基础介绍

了解视频解析平台API的基本概念和使用要求

核心 API 功能

视频解析平台现已简化并专注于以下核心功能：

  • 🔍 智能视频解析（主推）：
    • 唯一解析接口：/api/parse/smart
    • 自动尝试多个接口，失败后自动切换
    • 支持 api_key 参数，无需登录即可调用
    • 支持 type=video 直链播放页生成
  • 📋 接口列表查询：获取当前所有可用的解析接口（/api/parse/quick）
  • 🎬 影视导航目录：获取推荐的影视导航网站列表（/api/drama-sites）
  • 🔐 API Key 集成：完善的密钥认证体系，支持第三方集成和自动化调用
⚠️ 重要更新： 传统解析接口 /parse 已移除，建议所有新项目使用 /api/parse/smart 配合 api_key 参数。

认证要求

API 接口支持两种认证方式：

  • Session 登录（默认）：浏览器登录后直接调用，适合在平台内使用Web界面
  • API Key 认证（推荐）：适合外部调用、第三方集成和直链播放（如 type=video 播放页直链）
    • 在 用户中心 生成个人 API Key
    • 通过 URL 参数传递：?api_key=av_xxxxx（直接在API调用中使用）
    • 或通过请求头传递：X-API-Key: av_xxxxx 或 Authorization: Bearer av_xxxxx
    • API Key 错误时，接口将静默拒绝（403），不会给出任何提示
  • 部分接口需要管理员权限（如管理后台相关API）
🔐 API Key 使用建议：
  • 外部调用必须使用：如果没有有效的Session登录，必须通过 api_key 参数或请求头传递API Key
  • 优先使用URL参数：在智能解析等API调用中，直接在URL中添加 api_key 参数最为方便
  • 多平台集成：API Key是实现第三方集成的核心，可用于移动应用、桌面应用等任意平台
  • 直链播放页：通过 type=video&api_key=av_xxxxx 组合，可以直接生成播放页链接，无需登录即可观看
💡 智能解析 API Key 格式： /api/parse/smart?video_url=...&api_key=av_xxxxx&type=video

通用响应格式

所有JSON API接口都遵循统一的响应格式：

{
    "success": true,
    "message": "操作成功说明",
    "data": { /* 具体的返回数据 */ }
}

或失败时：

{
    "success": false,
    "message": "错误信息说明",
    "error": "具体的错误类型"
}

实际调用API接口的代码示例

JavaScript (Fetch API)

// ── 方式一：Session 登录（浏览器内调用）──
async function smartParseVideo(videoUrl) {
    const response = await fetch(`/api/parse/smart?video_url=${encodeURIComponent(videoUrl)}`);
    const data = await response.json();
    if (data.success) {
        document.getElementById('video-iframe').src = data.parse_url;
    }
}

// ── 方式二：API Key 外部调用（推荐）──
const API_KEY = 'av_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx'; // 从用户中心获取

// 1. 智能解析 + API Key（返回 JSON）
async function smartParseWithKey(videoUrl) {
    const url = `/api/parse/smart?video_url=${encodeURIComponent(videoUrl)}&api_key=${API_KEY}`;
    const data = await fetch(url).then(r => r.json());
    console.log("解析成功：", data.parse_url);
    return data.parse_url;
}

// 2. 智能解析 + API Key + 直链播放页（type=video）
function openVideoPage(videoUrl) {
    const url = `/api/parse/smart?type=video&video_url=${encodeURIComponent(videoUrl)}&api_key=${API_KEY}`;
    window.open(url, '_blank');
}

// 3. 完整参数智能解析（高级用法）
async function completeSmartParse(videoUrl, maxRetries = 3, startApiId = null) {
    let url = `/api/parse/smart?video_url=${encodeURIComponent(videoUrl)}&api_key=${API_KEY}&max_retries=${maxRetries}`;
    if (startApiId) {
        url += `&api_id=${startApiId}`;
    }
    const data = await fetch(url).then(r => r.json());
    console.log("尝试了以下接口：", data.tried_apis);
    return data;
}

// 4. 在 iframe 中嵌入播放页
function embedVideoInIframe(iframeId, videoUrl) {
    const url = `/api/parse/smart?type=video&video_url=${encodeURIComponent(videoUrl)}&api_key=${API_KEY}`;
    document.getElementById(iframeId).src = url;
}

Python (requests 库)

import requests
from typing import Optional, Dict, Any

BASE = 'http://your-domain'
API_KEY = 'av_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx'  # 从用户中心获取

def smart_parse_video(video_url: str, api_key: str = API_KEY, max_retries: int = 3,
                     start_api_id: Optional[int] = None, response_type: str = 'json') -> Optional[str]:
    """
    智能解析视频URL（使用API Key）
    :param video_url: 视频链接（必填）
    :param api_key: API密钥（必填，用于外部调用）
    :param max_retries: 最大尝试次数（默认3）
    :param start_api_id: 起始接口ID（可选）
    :param response_type: 响应类型 'json'（默认）或 'video'（直链播放页）
    :return: 解析成功的URL，失败返回None
    """
    params = {
        'video_url': video_url,
        'api_key': api_key,
        'max_retries': max_retries,
        'type': response_type
    }
    if start_api_id:
        params['api_id'] = start_api_id
    
    resp = requests.get(f'{BASE}/api/parse/smart', params=params)
    data = resp.json()
    
    if data.get('success'):
        if response_type == 'video':
            return resp.url  # 直链播放页URL
        else:
            return data['parse_url']  # JSON模式下的解析URL
    else:
        print(f"解析失败：{data.get('message', '未知错误')}")
        return None

# 获取播放页直链（无需登录，适合第三方嵌入）
def get_video_page_url(video_url: str, api_key: str = API_KEY) -> str:
    """
    生成视频播放页直链
    :param video_url: 视频链接
    :param api_key: API密钥
    :return: 可以直接在浏览器中打开的播放页URL
    """
    video_escaped = requests.utils.quote(video_url, safe='')
    return f'{BASE}/api/parse/smart?type=video&video_url={video_escaped}&api_key={api_key}'

# 高级用法：批量处理多个视频
def batch_parse_videos(video_urls: list, api_key: str = API_KEY) -> Dict[str, Optional[str]]:
    """
    批量解析多个视频
    :param video_urls: 视频链接列表
    :param api_key: API密钥
    :return: 字典 {视频URL: 解析后的URL}
    """
    results = {}
    for video_url in video_urls:
        try:
            parse_url = smart_parse_video(video_url, api_key)
            results[video_url] = parse_url
            print(f"✓ {video_url[:30]}... → {parse_url[:50] if parse_url else '失败'}...")
        except Exception as e:
            results[video_url] = None
            print(f"✗ {video_url[:30]}... → 异常: {e}")
    return results

# 快速调用示例
if __name__ == "__main__":
    # 示例1：JSON响应（获取解析URL）
    parsed_url = smart_parse_video("https://v.qq.com/x/cover/mzc002004qoigq3.html", API_KEY)
    if parsed_url:
        print(f"解析成功，播放链接: {parsed_url}")
    
    # 示例2：生成直链播放页（iframe嵌入）
    direct_link = get_video_page_url("https://v.qq.com/x/cover/mzc002004qoigq3.html", API_KEY)
    print(f"播放页直链: {direct_link}")
    
    # 示例3：批量解析
    video_list = [
        "https://v.qq.com/x/cover/abc123.html",
        "https://www.iqiyi.com/v_abc456.html"
    ]
    batch_results = batch_parse_videos(video_list, API_KEY)
    print(f"\n批量解析完成，成功: {sum(1 for v in batch_results.values() if v)} 个")

# ===============================
# 智能解析 API (推荐使用API Key)
# ===============================

# ── 1. 带有 API Key 的完整智能解析 ──
# 返回完整 JSON 响应（推荐开发调试）
curl -G "http://localhost:5000/api/parse/smart" \
  --data-urlencode "video_url=https://v.qq.com/x/cover/mzc002004qoigq3.html" \
  -d "api_key=av_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"

# 只查看解析结果（简洁格式）
curl -s "http://localhost:5000/api/parse/smart?video_url=https%3A%2F%2Fv.qq.com%2Fx%2Fcover%2Fmzc002004qoigq3.html&api_key=av_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx" | jq '.parse_url'

# 指定起始接口和最大尝试次数
curl -G "http://localhost:5000/api/parse/smart" \
  --data-urlencode "video_url=https://v.qq.com/x/cover/mzc002004qoigq3.html" \
  -d "api_key=av_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx" \
  -d "api_id=1" \
  -d "max_retries=5"

# ── 2. 获取直链播放页（type=video） ──
# 生成播放页链接（可在浏览器直接打开或嵌入iframe）
curl -s "http://localhost:5000/api/parse/smart?type=video&api_key=av_xxx&video_url=https%3A%2F%2Fv.qq.com%2F..." -L -I | head -5

# 直接打开（将链接复制到浏览器）
# http://localhost:5000/api/parse/smart?type=video&api_key=av_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx&video_url=https%3A%2F%2Fv.qq.com%2F...

# ── 3. 使用请求头的 API Key ──
# 通过 X-API-Key 请求头传递
curl -G "http://localhost:5000/api/parse/smart" \
  --data-urlencode "video_url=https://v.qq.com/x/cover/mzc002004qoigq3.html" \
  -H "X-API-Key: av_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"

# 通过 Authorization 请求头传递
curl -G "http://localhost:5000/api/parse/smart" \
  --data-urlencode "video_url=https://v.qq.com/x/cover/mzc002004qoigq3.html" \
  -H "Authorization: Bearer av_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"

# ===============================
# 辅助 API (可用 API Key 或 Session)
# ===============================

# ── 4. 获取所有解析接口列表 ──
# 使用 API Key
curl -G "http://localhost:5000/api/parse/quick" \
  -d "api_key=av_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"

# 使用 Session（需已登录）
curl -G "http://localhost:5000/api/parse/quick" \
  -H "Cookie: session=YOUR_SESSION_COOKIE"

# ── 5. 获取影视导航列表 ──
# 使用 API Key
curl -G "http://localhost:5000/api/drama-sites" \
  -d "api_key=av_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"

# 使用 Session
curl -G "http://localhost:5000/api/drama-sites" \
  -H "Cookie: session=YOUR_SESSION_COOKIE"

# ===============================
# 错误处理与调试
# ===============================

# 错误：缺少 API Key
curl -s "http://localhost:5000/api/parse/smart?video_url=https%3A%2F%2Fv.qq.com%2F..." | jq '.'

# 错误：无效 API Key（静默返回403）
curl -s "http://localhost:5000/api/parse/smart?video_url=https%3A%2F%2Fv.qq.com%2F...&api_key=invalid_key" | jq '.'

# 详细输出（可用于调试）
curl -v -G "http://localhost:5000/api/parse/smart" \
